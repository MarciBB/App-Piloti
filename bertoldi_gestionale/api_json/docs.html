API REST – Documentazione

Tutte le API sono esposte dal router /api_json/api.php e richiedono il parametro query token per l’autenticazione, ad esempio /api_json/api.php?handler=status&token=YOUR_TOKEN.

Scarica la collezione Postman: bertoldi_api_postman_collection.json. Imposta le variabili baseUrl e token prima di eseguire le richieste.

Specifica OpenAPI (importabile in Scrubby): openapi.json.

Autenticazione

Ogni chiamata deve includere token come parametro query. Il token è fornito agli operatori autorizzati all’uso dell’API ed è verificato contro la tabella ApiToken. Per alcuni endpoint (es. getBookings) è richiesto che l’utente appartenga al GestoreId = 1. Per ulteriori informazioni o per richiedere un token, rivolgersi a Bertoldi Boats.

Status

GET /api_json/api.php?handler=status&token={token}

Descrizione

Verifica la raggiungibilità del servizio e restituisce versione e stato.

Risposta

{
  "versione": "1.1.0",
  "status": "OK"
}

Experiences (lista)

GET /api_json/api.php?handler=experiences&token={token}[&tipoTour=0|1]

Descrizione

Restituisce l’elenco delle esperienze commercializzate. Il parametro opzionale tipoTour filtra per 0: Gruppo o 1: Privato.

Risposta

[
  {
    "LineaId": 12,
    "LineaNome": "Mini-Crociera Arcipelago",
    "TipoTour": "Tour di gruppo"
  }
]

Experience (dettaglio)

GET /api_json/api.php?handler=experience/{lineaId}&token={token}

Descrizione

Dettaglio di una singola esperienza/linea.

Risposta

{
  "LineaId": 12,
  "PercorsoNome": "Arcipelago Tour",
  "LineaNome": "Mini-Crociera Arcipelago",
  "TipoTour": "Tour di gruppo",
  "Porto": "Olbia",
  "Descrizione": "Itinerario giornaliero con soste...",
  "Link": "https://example.com/mini-crociera",
  "AndataRitorno": 1
}

Il campo AndataRitorno indica se per la linea sono disponibili corse di ritorno: 0 assente, 1 presente.

Experience Prices

GET /api_json/api.php?handler=experiencePrices&lineaId={id}&dataPartenza=YYYY-MM-DD&token={token}

Descrizione

Elenca le tipologie di biglietto disponibili per una data esperienza e data.

Risposta

[
  { "TipologiaBigliettoId": 17, "Nome": "Adulto", "Descrizione": "Passeggero adulto" },
  { "TipologiaBigliettoId": 18, "Nome": "Bambino", "Descrizione": "Passeggero 4-12 anni" }
]

Reservation

POST /api_json/api.php?handler=reservation&token={token}

Body (JSON)

{
  "departureDate": "2026-04-01",
  "departureRide": 345,
  "returnDate": null,
  "returnRide": null,
  "departureStationCode": 1,
  "arrivalStationCode": 2,
  "passengers": [
    { "Nome": "Mario", "Cognome": "Rossi", "TipologiaBiglietto": "Adulto", "TipologiaBigliettoId": 17 }
  ],
  "phoneNumber": "+390123456789",
  "email": "mario.rossi@example.com",
  "customerName": "Mario Rossi"
}

Descrizione

Crea una prenotazione per la/e corsa/e indicata/e con i passeggeri forniti. Per linee con andata/ritorno è possibile prenotare anche il ritorno impostando returnDate e returnRide usando una CorsaId tra quelle restituite in ridesReturn di availableRides.

Risposta

{
  "status": "ok",
  "PrenotazioneId": 789
}
In caso di esito positivo, usare l’endpoint booking per confermare/emissione titoli e ottenere i ticket.

Cancellazione

DELETE /api_json/api.php?handler=reservation/{prenotazioneId}&token={token}

Get Booking

GET /api_json/api.php?handler=booking/{codicePrenotazione}&token={token}

Restituisce la stessa struttura di booking dopo la conferma, includendo segmenti con typeExperience e experience e i dettagli ticket.

Cancellazione

DELETE /api_json/api.php?handler=booking/{codicePrenotazione}&token={token}[&force=true]

Se force=true ed entro la finestra temporale consentita, annulla e restituisce l’importo rimborsato in centesimi.

Get Bookings

GET /api_json/api.php?handler=getBookings&token={token}[&offset=n&limit=n]

Autorizzazione

Richiede che il token appartenga ad un operatore con GestoreId = 1; in caso contrario restituisce HTTP 403 e messaggio “Permessi non validi.”

Descrizione

Elenco delle prenotazioni con principali metadati (stato, linea, orari, importi, metodo pagamento, azienda emittente, ecc.). Supporta paginazione tramite offset e limit.

Risposta

[
  {
    "PrenotazioneId": 789,
    "PrenotazioneCodice": "ABC123",
    "PrenotazioneStato": "Confermata",
    "Agenzia": "Agenzia Demo",
    "DataCreazione": "01/04/2026 12:34:56",
    "UtenteCreatore": "operatore1",
    "LineaNome": "Mini-Crociera Arcipelago",
    "CorsaDataPartenza": "01/04/2026",
    "CorsaOrarioPartenza": "08:30",
    "ComuneSalita": "Olbia",
    "ComuneDiscesa": "La Maddalena",
    "TipoTour": "Gruppo",
    "TotalePostiPrenotati": 3,
    "TotalePrenotazione": 120.00,
    "Imbarcazione": "Motonave XYZ",
    "Provvigioni": 0,
    "Imponibile": 114.29,
    "IVA": 5.71,
    "PercentualeIVA": "5%",

Note e convenzioni

  • Date accettate: YYYY-MM-DD o DD/MM/YYYY dove previsto.
  • Prezzi in centesimi nei campi ...InCents.
  • I segmenti viaggio includono typeExperience (nome del Percorso) e experience (nome della Linea).
